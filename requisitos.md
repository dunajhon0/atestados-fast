# Prompt para generar un PRD de página web moderna, atractiva y monetizable con banners

Actúa como **Product Manager Senior + UX/UI Strategist + CRO Specialist + experto en monetización web**.

Tu tarea es crear un **PRD (Documento de Requisitos de Producto)** completo, profesional y accionable para desarrollar una página web con estas características:

## OBJETIVO GENERAL

- Página web visualmente atractiva, moderna y sencilla.
- Que invite al usuario a quedarse, explorar y usarla (alta retención/engagement).
- Que presente la información de forma interesante, clara y dinámica.
- Que sea monetizable mediante banners publicitarios (sin arruinar la experiencia de usuario).
- Que esté pensada para escalar y mejorar con iteraciones futuras.

## CONTEXTO DEL PROYECTO

- **Tipo de web:** [indica aquí la temática: blog / herramienta / landing / directorio / contenido educativo / nicho específico]
- **Público objetivo:** [describe edad, intereses, país, nivel técnico]
- **Objetivo de negocio principal:** [tráfico / leads / registros / ingresos por banners / afiliación / comunidad]
- **Dispositivo prioritario:** móvil (mobile-first), sin descuidar escritorio
- **Idioma principal:** español (España)
- **Estilo deseado:** elegante, limpio, moderno, llamativo pero no recargado

## QUIERO QUE EL PRD INCLUYA (OBLIGATORIO)

1. **Resumen ejecutivo del producto**
2. **Problema que resuelve y propuesta de valor**
3. **Objetivos del negocio y objetivos del usuario**
4. **Público objetivo / buyer persona** (con perfiles claros)
5. **Casos de uso principales y user journeys**
6. **Funcionalidades del MVP** (priorizadas: imprescindible / recomendable / futuro)
7. **Arquitectura de la información** (secciones, navegación, jerarquía de contenidos)
8. **Estrategia de contenido:**
   - cómo presentar la información para que resulte interesante
   - bloques visuales, titulares, microcopy, CTAs
   - orden recomendado de la información en página
9. **Requisitos de diseño UX/UI:**
   - estilo visual
   - principios de diseño
   - legibilidad
   - contraste
   - accesibilidad
   - consistencia
   - diseño mobile-first
10. **Requisitos de monetización con banners:**
   - ubicaciones sugeridas (sin ser intrusivas)
   - límites para no perjudicar UX
   - equilibrio contenido/publicidad
   - buenas prácticas para evitar caída de retención
   - recomendaciones para CTR sin engaños
   - cumplimiento de políticas publicitarias (genérico)
11. **Requisitos técnicos:**
   - rendimiento (carga rápida, Core Web Vitals)
   - SEO técnico básico
   - responsive
   - analítica (eventos clave)
   - estructura escalable
12. **Requisitos legales y de confianza:**
   - privacidad, cookies, aviso legal (a nivel general)
   - transparencia publicitaria
   - elementos de credibilidad
13. **KPIs y métricas de éxito:**
   - tiempo en página
   - rebote
   - páginas por sesión
   - CTR banners
   - RPM/ingresos estimables
   - conversión (si aplica)
14. **Riesgos y errores comunes a evitar**
15. **Roadmap por fases:**
   - Fase 1 (MVP)
   - Fase 2 (optimización UX/CRO)
   - Fase 3 (escalado)
16. **Criterios de aceptación del producto** (checklist verificable)
17. **Backlog inicial de tareas** (ordenado por prioridad)
18. **Sugerencias de tests A/B** para mejorar uso y monetización

## FORMATO DE SALIDA (MUY IMPORTANTE)

- Escríbelo en formato PRD profesional, con títulos y subtítulos claros.
- Usa tablas cuando ayuden (por ejemplo: funcionalidades, KPIs, roadmap, backlog).
- Sé específico, accionable y realista (nada genérico).
- Si falta información, haz supuestos razonables y márcalos como “Supuestos”.
- Incluye una versión final orientada a que pueda entregarse a un diseñador y a un desarrollador sin ambigüedades.

## ENFOQUE DE CALIDAD

- Prioriza experiencia de usuario + monetización sostenible (no agresiva).
- La web debe sentirse moderna, atractiva y útil, no una “web de anuncios”.
- El diseño y el contenido deben incitar al uso mediante claridad, utilidad, estética y buena estructura (sin tácticas engañosas).
