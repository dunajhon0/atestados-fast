## REQUISITOS OBLIGATORIOS SOBRE GESTIÓN DE IMÁGENES (MUY IMPORTANTE)
Incluye una subsección específica y detallada con TODOS estos puntos:

A) Gestión de rutas
- El sistema deberá permitir la carga de imágenes mediante rutas relativas al directorio del proyecto siempre que sea posible.
- El sistema deberá soportar rutas configurables para recursos estáticos (base path/base URL).
- El sistema no deberá depender de rutas absolutas locales del sistema operativo.
- El sistema deberá evitar enlaces hardcodeados a dominios o rutas locales.
- Estructura estándar de assets:
  /assets
     /images
     /css
     /js
- Convenciones de nombres (minúsculas, sin espacios, guiones, extensiones válidas).

B) Independencia del entorno (local / preview / staging / producción)
- Funcionar en local y producción sin cambiar manualmente rutas.
- Base URL configurable por entorno.
- Considerar diferencias Linux (case-sensitive).

C) Validación y prevención de errores
- Validación de existencia de archivos en build (CI) o predeploy.
- Reporte de rutas inválidas.
- Fallback (imagen alternativa) ante fallo.
- Registro de errores de carga si aplica.

D) Buenas prácticas
- Lazy loading cuando aplique.
- Formatos modernos (WebP/AVIF) con fallback.
- Compresión/optimización.
- `alt` accesible.
- Evitar CLS.

E) Prevención de errores comunes
- Control mayúsculas/minúsculas.
- Extensiones permitidas.
- Normalización de nombres problemáticos.
- Prohibición de referencias a imágenes inexistentes.

Incluye **10-15 requisitos formales (RF/RNF)** solo para imágenes/assets.
